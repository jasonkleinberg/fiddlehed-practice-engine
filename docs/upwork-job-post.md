## Job description (paste this)

I run an online fiddle school and I'm building a play-along practice tool. I have a library of fiddle **melodies engraved in Sibelius** (`.sib` files) that I need converted to **MusicXML**, and many of them need **simple chord symbols** added.

This is clean, well-defined work with an objective spec and an automated quality check — no guesswork about whether it's "done."

**Scope:** ~36 tunes (about 60 `.sib` files including some duplicate versions). Single-line fiddle melodies, short (most are 1–2 pages).

**Two phases:**
1. **Export** every `.sib` to uncompressed MusicXML 4.0 (exact settings provided). Mechanical.
2. **Add chord symbols** to the tunes that don't already have them — simple diatonic I–IV–V folk harmony, one chord per measure as a rule. I'll send the exact list after Phase 1.

You'll get a one-page instruction sheet with precise export settings, naming conventions, chord-style guidelines, worked examples, and the acceptance criteria. All files are delivered via a shared Dropbox folder.

**You're a fit if you:**
- Own and know **Sibelius** (MusicXML export, chord-symbol entry).
- Can read music and harmonize a simple folk melody with basic chords.
- Are careful with file naming and instructions.

You do NOT need to be an arranger — simple, tasteful triads are exactly right.

**Deliverables:**
- Phase 1: all `.sib` exported to `.musicxml`, same filenames, in the shared folder.
- Phase 2: chord symbols added to the listed tunes and re-exported.

**Acceptance:** an automated check verifies every file parses, has chord symbols where required, and has the correct title. Milestones release on passing that check.
